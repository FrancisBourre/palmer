Classpublic class Batch
InheritanceBatch AbstractCommand Object
Implements MacroCommand, CommandListener
Subclasses ContextParser

Batch object encapsulate a set of Commands to execute.

Default, Batch is a first-in first-out stack (FIFO) where commands are executed in the order they were registered. Batch can be LIFO using Batch instanctiation 2nd argument.

Default, the Event object received in the execute is passed to each commands contained in this batch. But you can relay event from command to command using Batch instanciaton first argument.

The Batch class extends AbstractCommand and so, dispatch an onCommandEnd event at the execution end of all commands.

protected var _bReversed:Boolean

Defines command list execution order.

Default is FIFO mode (false). Sets to true to use LIFO mode.

The default value is false.

_bUseEventFlowproperty 
protected var _bUseEventFlow:Boolean

Indicates if Batch use command event flow or only source event data.

If true next command is executed using last command event result. If false, only use the main source event data passed-in Batch.execute() call.

The default value is false.

process()method 
public static function process(f:Function, a:Array, ... args):void

Takes all elements of an Array and pass them one by one as arguments to a method of an object. It's exactly the same concept as batch processing in audio or video software, when you choose to run the same actions on a group of files.

Basical example which sets _alpha value to 0.4 and scale to 0.5 on all MovieClips nested in the Array :

Parameters

f:Function — function to run.
 
a:Array — array of parameters.
 
... args — additionnal parameters to concat with the passed-in arguments array


Example 
         import net.pixlib.commands.
                  function changeAlphaAndScale( mc : MovieClip, a : Number, s : Number )
         {
              mc.alpha = a;
              mc.scaleX = mc.scaleY = s;
         }
                  Batch.process( changeAlphaAndScale, [mc0, mc1, mc2], .4, 0.5 );
